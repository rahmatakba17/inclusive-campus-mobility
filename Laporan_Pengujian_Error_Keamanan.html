1 Pendahuluan & Lingkup Pengujian

Dokumen ini merupakan panduan teknis pengujian sistem Bus Kampus Non-Merdeka yang mencakup dua area utama: (1) Pengujian tampilan halaman error HTTP dan (2) Pengujian keamanan aplikasi web. Pengujian dilakukan pada lingkungan development lokal sebelum deployment ke production.

🌐 Informasi Sistem

FrameworkLaravel 10.x
DatabaseMySQL 8.0
AuthSession-based + CSRF
Base URLhttp://localhost:8181

👥 Role Pengguna

AdminAkses penuh sistem
UserPemesanan tiket
SopirDashboard perjalanan
GuestLihat bus (tanpa login)
⚠️ Perhatian: Route pengujian error (/test-error/*) hanya tersedia saat APP_ENV=local. Route ini tidak akan muncul di environment production/staging secara otomatis.

4 Pengujian Keamanan — CSRF Protection

Laravel melindungi semua form POST/PATCH/DELETE dengan CSRF token. Berikut cara mengujinya:

4.1 Skenario Pengujian CSRF

IDSkenarioLangkahHasil DiharapkanStatus
CSRF-01 Form Tanpa Token Submit form login via curl tanpa menyertakan _token HTTP 419 Page Expired ☐
CSRF-02 Token Kadaluarsa Buka form, tunggu >120 menit, submit HTTP 419 Page Expired ☐
CSRF-03 Token Salah Submit form dengan nilai _token dipalsukan HTTP 419 Page Expired ☐
CSRF-04 Cross-Origin Request Buat HTML form di domain lain mengarah ke /login HTTP 419, request ditolak ☐
CSRF-05 Token Valid — Form Normal Submit form login normal via browser Berhasil login (token diverifikasi Laravel) ☐

4.2 Cara Menguji CSRF dengan cURL

# Uji POST tanpa CSRF token — harus ditolak dengan 419
curl -X POST http://localhost:8181/login \
  -d "email=admin@kampus-non-merdeka.ac.id&password=password" \
  -c /tmp/cookies.txt -b /tmp/cookies.txt \
  -v 2>&1 | grep "HTTP/"

# Hasil yang diharapkan: HTTP/1.1 419

5 Pengujian Keamanan — SQL Injection

Laravel menggunakan Eloquent ORM dengan prepared statements yang mencegah SQL Injection secara default. Pengujian ini memverifikasi bahwa tidak ada query mentah yang rentan.

5.1 Skenario SQL Injection

IDInput UjiField TargetHasil DiharapkanStatus
SQLi-01 ' OR '1'='1 Email pada form login Login gagal, tidak bypass autentikasi ☐
SQLi-02 admin'-- Email pada form login Login gagal, error validasi ☐
SQLi-03 '; DROP TABLE users;-- Field pencarian / input teks Input diperlakukan sbg string biasa, tabel tidak terhapus ☐
SQLi-04 1 UNION SELECT username,password FROM users URL parameter ID booking Error 404 atau data tidak bocor ☐
SEC-05 update(['is_boarded'=>true]) Mass Assignment Exploit ($fillable) Sistem simulasi ORM menolak mass-assignment gelap (*Ghost-drop* dihiraukan jika tidak di-whitelist) ☐
✅ Proteksi Aktif: Semua query menggunakan Eloquent ORM (User::where('email', $email)) yang secara otomatis menggunakan parameterized queries / prepared statements — immune terhadap SQL Injection standar. Penutupan Celah Mass-Assignment Vulnerability di Model Booking (variabel is_boarded, priority_need) telah rampung sehingga eksploitasi oleh bot Auto-Cancel 15 Detik sudah dinetralisir selamanya.

6 Pengujian Keamanan — XSS (Cross-Site Scripting)

Laravel Blade menggunakan {{ }} (double curly) yang secara otomatis meng-escape HTML output. Pengujian ini memverifikasi tidak ada celah XSS reflected maupun stored.

6.1 Skenario XSS

IDPayloadTarget InputHasil DiharapkanStatus
XSS-01 <script>alert(1)</script> Field nama saat registrasi Tag ditampilkan sebagai teks biasa, alert TIDAK muncul ☐
XSS-02 <img src=x onerror=alert(1)> Field komentar / catatan booking Tag di-escape, tidak ada eksekusi JS ☐
XSS-03 "><svg onload=alert(1)> Parameter URL (reflected) Input di-encode, tidak ada eksekusi ☐
XSS-04 javascript:alert(1) Field URL / redirect Input ditolak atau ditampilkan sbg teks ☐

6.2 Verifikasi Otomatis Blade Escaping

Periksa source HTML halaman profil setelah menyimpan nama dengan karakter spesial:

# Input: nama = <script>alert('xss')</script>
# Output di HTML yang diharapkan:
<p>&lt;script&gt;alert('xss')&lt;/script&gt;</p>
# — Karakter < > dikonversi ke &lt; &gt; — AMAN ✅
⚠️ Perhatikan: Jika ada penggunaan {!! !!} (unescaped output) di Blade views, pastikan input yang ditampilkan sudah dibersihkan dengan strip_tags() atau htmlspecialchars().

7 Pengujian Keamanan — Rate Limiting & Brute Force

7.1 Uji Rate Limiting Form Login

IDSkenarioLangkahHasil DiharapkanStatus
RATE-01 Login Gagal Berulang Submit login salah 5x berturut-turut Akun dikunci sementara / throttle aktif (429 Too Many Requests) ☐
RATE-02 API Endpoint Flooding Kirim 100+ request ke /api/simulation/buses dalam 1 menit Rate limiter memblokir setelah batas terlampaui ☐
RATE-03 Password Enumeration Coba 50+ password berbeda untuk 1 email Respon terlambat / diblokir setelah percobaan tertentu ☐

7.2 Uji Paksa dengan cURL (Simulasi Brute Force)

# Simulasi login gagal berulang — pantau apakah 429 muncul setelah percobaan ke-5
for i in {1..6}; do
  curl -s -o /dev/null -w "Request $i: HTTP %{http_code}\n" \
    -X POST http://localhost:8181/login \
    -H "Content-Type: application/x-www-form-urlencoded" \
    -d "_token=INVALID&email=test@test.com&password=wrongpass"
done

# Hasil diharapkan: Request 1-5 = 422, Request 6 = 429 Too Many Requests

8 Checklist Aksesibilitas WCAG 2.1

Bus Kampus Non-Merdeka telah mengimplementasikan standar WCAG 2.1 Level AA. Gunakan checklist ini untuk memverifikasi:

🔑 Keyboard & Navigasi

  • Skip link "Lewati ke konten utama" muncul saat Tab pertama ditekan
  • Semua tombol dan link dapat dijangkau via Tab
  • Focus indicator (ring merah 3px) terlihat jelas
  • Modal panel aksesibilitas ditutup dengan tombol Escape
  • Sidebar navigation bisa diakses penuh via keyboard

🎨 Kontras & Visual

  • Teks sidebar: rasio kontras ≥ 4.5:1 (rgba white/0.85)
  • Badge status: warna teks memenuhi WCAG AA
  • Tombol logout memiliki label teks, bukan hanya ikon
  • Ikon FontAwesome dekoratif: aria-hidden="true"
  • High contrast mode berfungsi via toolbar aksesibilitas

📢 Screen Reader

  • Semua gambar punya alt text deskriptif
  • Flash message: role="alert" aria-live="polite"
  • Link aktif sidebar: aria-current="page"
  • Mobile menu button: aria-expanded + aria-controls
  • Rating bintang: aria-label="Rating 5 bintang"

⚙️ Aksesibilitas Toolbar

  • Tombol ♿ terfokus saat Tab ke wilayahnya
  • Toggle font A/A+/A++ mengubah ukuran teks situs
  • Toggle kontras tinggi aktif dan persisten
  • Toggle kurangi gerak menonaktifkan animasi
  • Reset mengembalikan semua ke default

8.1 Cara Menguji Aksesibilitas

MetodeToolLangkah
Keyboard-only Browser saja Tekan Tab berulang kali, navigasi tanpa mouse. Pastikan semua elemen tercapai
Color Contrast Chrome DevTools Inspect element → klik kotak warna → lihat rasio kontras otomatis
Screen Reader NVDA (Windows) / VoiceOver (Mac) Aktifkan, navigasi halaman, pastikan semua konten diumumkan
Automated Scan axe DevTools (Chrome Extension) Install extension → buka halaman → Run Analysis → periksa violations
Reduced Motion OS Settings / Toolbar Aktifkan "Reduce Motion" di OS atau via toolbar ♿ → verifikasi animasi berhenti

9 Ringkasan & Rekomendasi

9.1 Status Proteksi Keamanan

AreaStatusMekanismeRekomendasi
CSRF ✅ Aktif Laravel VerifyCsrfToken middleware Sudah aman — tidak perlu aksi
SQL Injection ✅ Aktif Eloquent ORM + prepared statements Hindari query DB::select() dengan input mentah
XSS ✅ Aktif Blade {{ }} auto-escape HTML Hindari {!! !!} tanpa sanitasi
Autentikasi ✅ Aktif Session-based auth + middleware auth Pertimbangkan 2FA untuk akun admin
Otorisasi ✅ Aktif Custom middleware: admin, sopir Tambahkan Laravel Gates/Policies untuk granularitas
Rate Limiting ⚠️ Parsial Laravel default throttle Konfigurasi eksplisit di RouteServiceProvider
HTTPS ⚠️ Lokal HTTP di development Wajib SSL/TLS di production (Let's Encrypt)
Headers Keamanan ⚠️ Belum — Tambahkan CSP, X-Frame-Options, HSTS via middleware

9.2 Rekomendasi Prioritas Tinggi

🔴 PRODUCTION — Wajib Sebelum Go-Live:
  1. Aktifkan HTTPS / SSL certificate (Let's Encrypt)
  2. Set APP_DEBUG=false dan APP_ENV=production
  3. Tambahkan header keamanan: X-Frame-Options: DENY, Content-Security-Policy
  4. Hapus atau batasi akses route /test-error/* (sudah terlindungi via app()->environment('local'))
  5. Konfigurasikan SESSION_SECURE_COOKIE=true dan SESSION_HTTP_ONLY=true
🟡 JANGKA MENENGAH — Peningkatan Keamanan:
  1. Implementasi Two-Factor Authentication (2FA) untuk admin
  2. Audit log untuk aksi sensitif (login, perubahan data, hapus record)
  3. Konfigurasi rate limiting eksplisit di RouteServiceProvider.php
  4. Tambahkan validasi server-side yang lebih ketat di semua FormRequest
🟢 AKSESIBILITAS — Sudah Diimplementasikan:
  • Skip navigation link (WCAG 2.4.1)
  • ARIA landmark roles di semua layout (WCAG 1.3.1)
  • Focus indicator yang jelas (WCAG 2.4.7)
  • Accessibility Toolbar: ukuran teks, kontras, reduced motion
  • Mendukung prefers-reduced-motion dan Windows High Contrast Mode

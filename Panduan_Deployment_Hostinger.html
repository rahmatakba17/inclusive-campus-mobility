Panduan Lengkap: Deployment Bus Kampus di Hostinger (Metode SSH & Git)

Panduan ini menggunakan metode Terminal (SSH) dan Git untuk mempermudah proses pembaruan (update) di masa depan tanpa perlu upload manual berkali-kali.

Penting: Pastikan Anda telah melakukan git commit dan git push seluruh kode proyek Bus Kampus lokal Anda (termasuk folder file tersembunyi) ke repositori GitHub atau GitLab Anda secara online.

Tahap 1: Persiapan Database di Hostinger

  1. Login ke hPanel Hostinger.
  2. Buka menu Database -> Manajemen Database (MySQL Databases).
  3. Buat database baru. Isi Nama Database, Username MySQL, dan Password.
  4. Klik tombol Buat (Create), dan catat informasi kredensial yang baru saja Anda buat.

Tahap 2: Menghubungkan Terminal via SSH

  1. Di hPanel Hostinger, cari menu Tingkat Lanjut (Advanced) -> Akses SSH.
  2. Aktifkan akses SSH jika belum aktif.
  3. Salin perintah koneksi SSH yang diberikan (misal: ssh u1234567@185.xxx.xxx.xxx -p 65002).
  4. Buka program Terminal (di Mac/Linux) atau Command Prompt/PowerShell (di Windows), lalu *paste* dan jalankan perintah tersebut.
  5. Masukkan password FTP/SSH Anda (biasanya sama dengan password utama Cpanel/hPanel Anda). Ketika mengetik, karakter password tidak akan terlihat; tekan Enter jika sudah.

Tahap 3: Kloning Repositori Git di Server

Setelah berhasil login di Terminal Hostinger:

  1. Masuk ke direktori website Anda (biasanya `public_html`): 
    cd domains/namadomainanda.com/public_html
  2. Hapus file bawaan jika ada (memastikan folder kosong sempurna): 
    rm -rf * && rm -rf .*
  3. Lakukan *Clone* (Salin kode) dari repositori GitHub Anda ke dalam direktori ini (jangan lupa tanda titik di akhir agar terekstrak langsung, bukan di dalam sub-folder): 
    git clone https://github.com/username/repo-bus-kampus.git .

Tahap 4: Instalasi Dependensi dan Build

Jalankan perintah ini satu-per-satu di terminal VPS/Shared Hosting Anda:

  1. Install berbagai library (vendor) PHP: 
    composer install --no-dev --optimize-autoloader
  2. Install Node.js packages untuk bagian Frontend (Asset): 
    npm install
npm run build

Tahap 5: Konfigurasi File .env dan Migrate

  1. Buat file .env dengan mengkopi file example: 
    cp .env.example .env
  2. Edit file .env menggunakan editor Nano: 
    nano .env

    Ubah pengaturan krusial menjadi seperti ini (Gunakan tombol panah di keyboard untuk navigasi arah):

    APP_ENV=production
APP_DEBUG=false
APP_URL=https://namadomainanda.com

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=nama_database_anda_dari_tahap_1
DB_USERNAME=user_database_anda_dari_tahap_1
DB_PASSWORD=PasswordDatabaseAnda!

    Simpan konfigurasi dengan menekan Ctrl + X, lalu ketik huruf Y, kemudian tekan Enter.

  3. Generate Key enkripsi lokal: 
    php artisan key:generate
  4. Jalankan pembuatan tabel database (Migrasi) beserta isinya: 
    php artisan migrate --seed

Tahap 6: Optimasi & Mengamankan Folder Utama

  1. Tautkan folder penyimpanan publik gambar/assets Laravel (Symlink storage): 
    php artisan storage:link
  2. Optimasi agar Laravel memuat kerangka aplikasi lebih cepat di Production: 
    php artisan optimize
  3. Tahap Akhir (Sangat Penting untuk Shared Hosting!): Secara default aplikasi hanya mengekspos public folder. Buat sebuah file konfigurasi server htaccess: 
    nano .htaccess
    Tambahkan kode teks berikut ini: 
    <IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteRule ^(.*)$ public/$1 [L]
</IfModule>
    (Simpan: Ctrl+X, Y, lalu Enter.)

Buka website dari browser (contoh: https://namadomainanda.com) untuk memastikan aplikasi telah mengudara!

Tahap Pembaruan (Jika Ada Perbaikan Bug / Penambahan Fitur Lokal)

Sewaktu-waktu di masa mendatang Anda merubah kode di komputer pribadi Anda, dorong (Push) perubahan tersebut ke Github. Lalu Login SSH dan jalankan rutinitas sederhana (Pulling):

cd domains/namadomainanda.com/public_html
git pull origin main
composer install --no-dev
npm run build
php artisan migrate
php artisan optimize

Sistem akan langsung terbarui secara cerdas tanpa harus upload manual (ZIP) file yang berat.
